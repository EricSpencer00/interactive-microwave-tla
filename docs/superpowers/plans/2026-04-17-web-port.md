# Interactive Microwave — Web Port & Stuttering Fix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a static, browser-native version of the TLA+ microwave simulator with a JS-native TLA+ checker that correctly models stuttering, and publish it as a blog post on ai4fm.cs.luc.edu.

**Architecture:** Standalone `web/` directory built with Vite + TypeScript + React. No server. The TLA+ runtime is a literal TS transcription of `Microwave.tla` plus an explicit `Stutter` action realizing `[Next]_vars`. Output bundle is copied into the ai4fm Sphinx repo and embedded in a new post under `src/posts/`.

**Tech Stack:** TypeScript, React 18, Vite, Vitest (tests), existing Spring Boot code left untouched for local dev parity.

```ts
describe('Microwave Next', () => {

    const successors = Next(Init);

    expect(tp?.v2.power).toBe('ON');

    expect(t?.v2.time).toBe(0);
    expect(t?.v2.radiation).toBe('OFF');
  });

  it('Safe rejects radiation ON with door OPEN', () => {
    expect(Safe({ door: 'OPEN', time: 5, radiation: 'ON', power: 'ON' })).toBe(false);
```

```ts
import { Vars, MAX_TIME } from '../model/state';

export interface Action { name: string; enabled(v: Vars): boolean; step(v: Vars): Vars; }

const TogglePower: Action = {
  name: 'TogglePower',
  enabled: () => true,
  step: v => ({ ...v, power: v.power === 'ON' ? 'OFF' : 'ON' }),
};

const IncrementTime: Action = {
  name: 'IncrementTime',
  enabled: v => v.time + 3 <= MAX_TIME,
  step: v => ({ ...v, time: v.time + 3 }),
};

const Start: Action = {
  name: 'Start',
  enabled: v => v.time > 0 && v.door === 'CLOSED' && v.power === 'ON',
  step: v => ({ ...v, radiation: 'ON' }),
};

const Tick: Action = {
  name: 'Tick',
  enabled: v => v.time > 0,
  step: v => {
    const t2 = v.time - 1;
    return { ...v, time: t2, radiation: t2 === 0 ? 'OFF' : v.radiation };
  },
};

const Cancel: Action = {
  name: 'Cancel',
  enabled: () => true,
  step: v => ({ ...v, time: 0, radiation: 'OFF' }),
};

const CloseDoor: Action = {
  name: 'CloseDoor',
  enabled: v => v.door === 'OPEN',
  step: v => ({ ...v, door: 'CLOSED' }),
};

const OpenDoor: Action = {
  name: 'OpenDoor',
  enabled: v => v.door === 'CLOSED',
  step: v => ({ ...v, door: 'OPEN', radiation: 'OFF' }),
};

export const Actions: Action[] = [TogglePower, IncrementTime, Start, Tick, Cancel, CloseDoor, OpenDoor];

export const Next = (v: Vars): { name: string; v2: Vars }[] =>
  Actions.filter(a => a.enabled(v)).map(a => ({ name: a.name, v2: a.step(v) }));

export const Safe = (v: Vars): boolean => !(v.radiation === 'ON' && v.door === 'OPEN');
```

```ts
import { Vars } from '../model/state';
import { Next, Safe } from './spec';

const key = (v: Vars) => `${v.door}|${v.time}|${v.radiation}|${v.power}`;

export const reachable = (init: Vars): Map<string, Vars> => {
  const seen = new Map<string, Vars>();
  const stack = [init];
  while (stack.length) {
    const v = stack.pop()!;
    const k = key(v);
    if (seen.has(k)) continue;
    seen.set(k, v);
    for (const { v2 } of Next(v)) stack.push(v2);
  }
  return seen;
};

export const findViolations = (init: Vars): Vars[] =>
  [...reachable(init).values()].filter(v => !Safe(v));
```

### Task 6: Simulation engine (click dispatch + tick loop with stutter emission)

**Files:**
- Create: `web/src/sim/engine.ts`

- [ ] **Step 1:** Write the engine: holds `Vars`, exposes `dispatch(actionName)`, runs a 1 s `setInterval` tick, and emits a `(stutter)` trace row when no action is auto-fired:

```ts
import { Vars, Init } from '../model/state';
import { Actions } from '../checker/spec';
import { Stutter } from '../checker/stutter';

export interface TraceRow { t: number; name: string; from: Vars; to: Vars; safe: boolean; }

export class Engine {
  private v: Vars = { ...Init };
  private trace: TraceRow[] = [];
  private listeners = new Set<() => void>();
  private tickHandle?: number;
  public showStutter = true;

  getState() { return this.v; }
  getTrace() { return this.trace; }
  subscribe(fn: () => void) { this.listeners.add(fn); return () => this.listeners.delete(fn); }
  private notify() { this.listeners.forEach(l => l()); }

  dispatch(name: string): boolean {
    const action = Actions.find(a => a.name === name);
    if (!action || !action.enabled(this.v)) return false;
    const from = this.v;
    const to = action.step(this.v);
    this.v = to;
    this.push({ name, from, to });
    this.notify();
    return true;
  }

  start() {
    if (this.tickHandle !== undefined) return;
    this.tickHandle = window.setInterval(() => this.onTick(), 1000);
  }
  stop() {
    if (this.tickHandle !== undefined) { clearInterval(this.tickHandle); this.tickHandle = undefined; }
  }

  private onTick() {
    // Auto-tick semantics: only fire Tick when radiation is ON; otherwise emit stutter
    const tick = Actions.find(a => a.name === 'Tick')!;
    if (this.v.radiation === 'ON' && tick.enabled(this.v)) {
      const from = this.v;
      this.v = tick.step(this.v);
      this.push({ name: 'Tick', from, to: this.v });
    } else {
      const from = this.v;
      const to = Stutter.step(this.v);
      this.push({ name: Stutter.name, from, to });
    }
    this.notify();
  }

  private push(row: Omit<TraceRow, 't' | 'safe'>) {
    const safe = !(row.to.radiation === 'ON' && row.to.door === 'OPEN');
    this.trace.push({ t: Date.now(), safe, ...row });
    if (this.trace.length > 500) this.trace.splice(0, this.trace.length - 500);
  }

  reset() { this.v = { ...Init }; this.trace = []; this.notify(); }
}
```

```rst
Interactive Microwave: TLA+ in the Browser
==========================================

.. post:: 2026-04-17
   :author: Eric Spencer
   :category: demos
   :tags: tla+, formal-methods, stuttering

A browser-native version of the `interactive-microwave-tla
<https://github.com/EricSpencer00/interactive-microwave-tla>`_ simulator. It
mimics a Java microwave runtime and a small TLA+ model checker faithful to the
``Microwave.tla`` spec, including explicit stuttering steps as required by
``Spec == Init /\ [][Next]_vars``.

The stuttering fix
------------------

The previous web demo never emitted a stutter step: every logged trace row was
a real action. Because ``[Next]_vars`` admits ``vars' = vars``, this was an
incomplete rendering of the spec's semantics. The new build adds an explicit
``Stutter`` action and logs it on every tick where no user action fires, so
behaviors faithfully alternate real transitions with stuttering.
```
